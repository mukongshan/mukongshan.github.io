Ch3-1 · 有穷自动机

确定有穷自动机（Deterministic Finite Automata

，DFA）

概述

  1. 是一个五元组

    1. 有限的状态集Q
    2. 有限的输入Σ
    3. 转换函数𝛿
    4. 初始状态q
    5. 终止状态集F

  2. 总能输出”accept”或”reject”

  3. 有两个“基本集合”：最终状态和非最终状态

Minimization

  1. 首先，把状态区分为最终状态和非最终状态两个集合
  2. 然后细分集合，把“去向”不同状态的分开

Bi-Simulation

  • 用于判断两个状态机是否等价
  1. 用一个pair把两者的起始状态表示出来
  2. 写出它们分别经过a, b后的pair
  3. 若这些pair没有出现在state pairs中，则加入进去，重复上述操作
  4. 如果两个状态机同时进入最终状态，则等价

不确定有穷自动机（Non-deterministic Finite Automata，NFA）

概述

  1. 判断：

    1. “路径”是不确定的，可能有多条路
    2. 含有空字符边

  2. 如果一个走不通，就试其他路径

  3. 只要有一条路通了，就认为这个自动机接受这个字符串

𝜖-Closure

  • 如果从状态集合 S 出发，可以经过零个或多个ε转换到达其他状态，那么这些状态都属于𝜖-Closure(S)。

$δ^*$

  • δ∗(q,w)={q′∣q′∈δ(q,a) for some a∈Σ or w=ϵ}
  • 用于描述在给定状态下，经过一整个输入字符串的处理后，自动机可能到达的状态集合。

DFA和NFA的关联

  1. NFA<=>DFA，二者可以相互转化

    1. DFA是特殊的NFA，只不过没有空串，且只有一条出边

    2. NFA可以通过子集构造法变为DFA

  2. 若一个集合中含有final state，则整个集合构成的状态是final state

  3. 子集构造法：

    1. 核心思想：让构造的DFA的每个状态对应NFA的一个状态集合
    2. 如果NFA中有n个状态，则DFA中最多可以有2^n^个状态。因为DFA中的状态是NFA状态集合的子集。

  4. L(a)表示a的自动机

Ch4-1 · 上下文无关语言

上下文无关文法（Context-Free Grammar，CFG）

  1. 定义
    1. 上下文无关文法（Context-Free Grammar，CFG）是一种用于描述计算机语言和自然语言语法的形式文法。它由四元组 G=(V,Σ,P,S)G = (V, \Sigma, P, S)G=(V,Σ,P,S) 组成，其中：
      1. 终结符集合 Σ：一组基础符号，不能再被替换（如字母、数字、标点等）。
      2. 非终结符集合 V：一组可以继续展开的符号（如语法成分：句子、名词短语等）。
      3. 规则集合 P：一组产生式规则，每条规则的形式为 A→α，其中 A 是非终结符，α 是终结符和/或非终结符的序列。
      4. 起始符号 S：语法生成的起点，通常是一个非终结符。
  2. 推导
    1. 最左推导和最右推导：就是优先把最左边或者最右边的符号展开
  3. 歧义性：
    1. 如果一个表达式可以构造出两棵语法树，则是有歧义的
    2. 例子：w = a + a * a.这是有歧义的

Chomsky Normal Form（CNF）

  1. Chomsky Normal Form (CNF, 乔姆斯基范式) 是上下文无关文法（Context-Free Grammar, CFG）的一种标准形式。若一个上下文无关文法符合 CNF，则它的所有产生规则必须满足以下两种形式之一：

    1. 二元规则（Binary Rules）：
      形如 A → BC，其中 A, B, C 均为非终结符（B 和 C 不能是起始符 S）。
    2. 终结规则（Terminal Rules）：
      形如 A → a，其中 A 为非终结符，a 为终结符。

    此外，还允许一个特殊的起始符规则：

    • S → ε（仅当 S 不出现在其他产生式的右侧时），表示空串。

Ch4-2 · 语法分析(Syntax Analysis)

自上而下的语法分析（Top-down parsing）

概述

  1. Top-down parsing 是一种解析技术，用于分析输入字符串的语法结构。它从语法的起始符号（根节点）开始，逐步向下扩展，直到生成与输入字符串匹配的解析树
  2. 特点：
    1. 解析树的构建方向：从根节点（起始符号）开始，逐步向下构建解析树。
    2. 推导方式：使用左推导（leftmost derivation），即每次替换最左边的非终结符。
    3. 解析过程：解析器尝试应用语法规则，逐步匹配输入字符串。如果匹配失败，可能需要回溯（backtracking）。
  3. 主要类型
    1. 递归下降解析（Recursive Descent Parsing）：
      • 通过递归函数实现解析，每个非终结符对应一个函数。
      • 可能需要回溯，尤其是在语法存在左递归或多个可选规则时。
    2. 预测解析（Predictive Parsing）：
      • 一种特殊的递归下降解析，通过向前看（lookahead）符号预测应用的规则。
      • 不需要回溯，效率更高，但对语法的要求更严格。

LL（1）文法

  1. 定义

    LL(1) 是一种 自上而下的预测分析文法，其中：

    • 第一个 L：表示 Left-to-right（从左到右解析）
    • 第二个 L：表示 Leftmost derivation（最左推导）
    • (1)：表示 每次只查看一个输入符号 来决定如何推导

  2. 特点：

    1. 不是二义性的 (Not ambiguous)：每个输入字符串最多有一个语法分析树。
    2. 没有左递归 (Not left-recursive)：文法不能包含直接或间接左递归，否则预测分析无法工作。
    3. 可以使用一个符号的前瞻 (1-token lookahead) 进行预测。

  3. LL(1) 文法的条件

    文法是 LL(1) 当且仅当 所有产生式$A→α∣βA \to \alpha | \beta A→α∣β$ 满足：

    1. α 和 β 不能以相同的终结符开头（避免选择冲突）。

Ch12-1 · Inter-procedural Data Flow Analysis

概述

  1. 交叉过程数据流分析是一种程序分析技术，用于分析程序中数据在不同过程（函数、方法）之间的流动。与仅分析单个过程的**内过程（intra-procedural）**数据流分析不同，交叉过程分析考虑了过程之间的调用关系，能够更全面地理解程序的行为和数据依赖关系

调用图和调用链

  1. 调用图是一个有向图，表示程序中各个函数之间的调用关系。每个节点代表一个函数，每条有向边表示一个函数调用了另一个函数。
  2. 调用链是从程序的入口点到某个特定函数的调用路径。调用链可以看作是调用图中的一条路径，表示函数调用的顺序。

上下文敏感的数据流分析（Context-Sensitive Data Flow Analysis）

  1. 基于克隆的交叉过程数据流分析（Clone-based Interprocedural DFA）

    1. 克隆过程：
      • 对于每个过程，根据其调用上下文创建多个副本。每个副本对应一个特定的调用路径。
      • 例如，如果一个过程被两个不同的调用者调用，那么该过程会被克隆为两个副本，每个副本对应一个调用者。

  2. 基于摘要的交叉过程数据流分析（Summary-based Inter-procedural DFA）

    1. 自顶向下（Top-Down, TD）分析

      自顶向下的分析方法从调用者（caller）开始，逐步分析被调用者（callee）。这种方法更具体地考虑了调用上下文，能够更精确地捕捉数据依赖关系。

    2. 自底向上（Bottom-Up, BU）分析

      自底向上的分析方法从被调用者（callee）开始，逐步分析调用者（caller）。这种方法需要假设所有可能的调用上下文，但更容易并行化。

Indexing Conventional Reachability

  1. 标签的定义

    标签通常是一个包含两个值的数组 [low, high]，其中：

    • low：表示子图中的最小值。
    • high：表示节点的后序遍历值（post-order value）。

  2. 标签的性质

    1. A → B ⇒ LabelB ⊆ LabelA：
      • 如果存在一条从节点 A 到节点 B 的路径，则节点 B 的标签必须是节点 A 标签的子集。
      • 这是因为在路径上，节点 B 的 low 值必须大于或等于节点 A 的 low 值，而 high 值必须小于或等于节点 A 的 high 值。
    2. LabelB ⊈ LabelA ⇒ A ↛ B：

      1. 为了判断一个文法是否是 LL(1) 文法，需要计算每个非终结符的 First 集合 和 Follow 集合：
        1. First(A)：A的展开式中最左的终结符集合。若A为终结符，则就是它本身
          1. First(A)要么是终结符，要么是𝜖
        2. Follow(A)：A后面紧跟的终结符集合，不包含A。注意，Follow只对非终结符有效。对于终结符不用写。

            2. 若$A\to{BD}$，则把First(D) \ {𝜖}加入Follow(B)
            3. 若$A\to{aB}$，注意，此时A的follow一定是B的follow，换言之Follow(A)是Follow(B)的子集。

          1. 对于 A→a 和 A→ϵ，First(a)={a} 和 First(ϵ)={ϵ} 互斥。

  1. 位移规约解析
    1. 位移规约解析是一种自底向上的语法分析方法，广泛应用于编译器设计中。它通过将输入符号逐步推入栈中，并根据文法规则进行规约，最终生成语法树。位移规约解析的核心在于两个基本操作：位移（Shift） 和 规约（Reduce）。
    2. 基本概念
      1. 位移（Shift）
        1. 操作：将输入符号从输入缓冲区移动到分析栈的栈顶。
        2. 目的：将输入符号逐步推入栈中，为后续的规约操作做准备。
      2. 规约（Reduce）
        1. 操作：根据文法规则，将栈顶的若干符号替换为规则的左侧非终结符。
        2. 目的：逐步构建语法树，将输入符号归约为更高层次的语法结构。
      3. 句柄：具体来说，句柄是当前栈顶符号序列中最左边的一个子序列，该子序列可以匹配某个产生式的右侧。
    3. 位移规约解析的步骤
      1. 初始化：
        • 创建一个分析栈，初始时栈中只有一个特殊的起始符号（如 $）。
        • 创建一个输入缓冲区，包含输入符号和输入结束符（如 $）。
      2. 分析过程：
        • 重复以下步骤，直到分析栈和输入缓冲区都为空：
          1. 位移操作：
            • 如果当前输入符号是一个终结符，并且根据当前状态和输入符号，应该进行位移操作，则将输入符号推入分析栈，并移动输入指针到下一个符号。
          2. 规约操作：
            • 如果分析栈的栈顶符号可以匹配某个产生式的右侧，则根据该产生式进行规约操作。将产生式右侧的符号从栈顶弹出，并将产生式左侧的非终结符推入栈中。
          3. 接受状态：
            • 如果分析栈中只剩下起始符号和输入结束符，且输入缓冲区为空，则解析成功。
          4. 错误处理：
            • 如果无法进行位移或规约操作，则报告语法错误。

  2. 函数调用时，栈中发生了什么（即函数调用的步骤）：

    1. . 调用前

      • 调用者的状态保存：在调用函数之前，调用者（即调用当前函数的代码）的执行状态需要被保存。这通常包括：
        • 寄存器保存：某些寄存器（如通用寄存器、状态寄存器等）的值会被保存到栈中。这些寄存器可能包含调用者的局部变量、状态信息等。
        • 返回地址保存：调用者的下一条指令地址（即函数调用完成后需要返回到的位置）会被保存到栈中。这个地址通常被称为返回地址。
        • 参数准备：调用者会将函数调用所需的参数准备好，并按照一定的顺序（通常是从右到左）压入栈中。这些参数将被传递给被调用的函数。

      2. 调用时

      • 栈帧创建：当函数被调用时，系统会为被调用的函数创建一个新的栈帧（Stack Frame）。栈帧是栈中的一个区域，用于存储函数调用期间所需的所有信息。
        • 局部变量分配：被调用函数的局部变量会在栈帧中分配空间。这些局部变量只在函数调用期间有效，函数调用结束后，这些空间会被释放。
        • 寄存器保存与恢复：某些寄存器的值可能会被保存到栈帧中，以便在函数调用期间使用这些寄存器。函数调用结束后，这些寄存器的值会被恢复。
        • 函数执行：被调用的函数开始执行，它会使用栈帧中的局部变量和参数进行计算。在函数执行过程中，可能会进行更多的函数调用，从而在栈中创建更多的栈帧。

      3. 调用后

      • 返回值处理：函数执行完成后，会将返回值（如果有）存储在某个约定的位置（通常是某个寄存器或栈中的某个位置）。
      • 栈帧销毁：被调用函数的栈帧会被销毁。这包括释放局部变量占用的空间，恢复之前保存的寄存器值等。
      • 返回地址恢复：从栈中恢复之前保存的返回地址，程序控制权会跳转到这个地址，继续执行调用者的代码。
      • 调用者状态恢复：调用者的寄存器值等状态信息被恢复，调用者的代码继续执行。

      总结

      • 调用前：保存调用者的状态，准备参数。
        • 例如：sw ra, 4(sp)把调用者的返回地址线保存到栈上。
      • 调用时：创建栈帧，分配局部变量空间，执行函数。
        • 例如：addi sp, sp, -4
      • 调用后：销毁栈帧，恢复调用者状态，继续执行调用者的代码。
        • 例如：恢复返回地址lw ra 0(sp)，销毁栈帧：addi sp, sp, 4

  • Datalog-based analysis 就是把静态分析的规则用 Datalog 写出来，交给逻辑引擎去执行，从而实现自动、高效的程序分析。
  1. 到达定义的DataLog表示法

    1. 三个谓词

      1. def(B, N, X)：在 基本块 B（Block B） 中，第 N 条语句（statement N） 定义了变量 X。
      2. succ(B, N, C)：按照PPT的说法，基本块C是基本块B的后继，且B有N条语句。
      3. rd(B, N, C, M, X)：变量 X 在基本块 C 的第 M 条语句中被定义，该定义 可以在控制流图中传播到 基本块 B 的第 N 条语句。

    2. 三个规则

      • “:-”的意思是从右向左推导，如果右边成立，那么左边就成立
      1. 定义即到达：
        1. rd(B, N, B, N, X) :- def(B, N, X)
        2. 定义可以到达它自己
      2. 语句内部前向传播（顺序语句）
        1. rd(B, N, C, M, X) :- rd(B, N-1, C, M, X), def(B, N, Y), X≠Y
        2. 如果定义 X 到达了语句 N-1，并且语句 N 没有重定义 X，那么 X 的定义也能到达语句 N。
      3. 跨基本块传播
        1. rd(B, 0, C, M, X) :- rd(D, N, C, M, X), succ(D, N, B)
        2. 如果 X 的定义在 (C, M) 能到达基本块 D 的语句 N，并且 D 的语句 N 有控制流边到达基本块 B，那么我们认为这个定义在 B 的“开始处”（第 0 行）可达。
        3. 第 0 行是一个虚拟位置，代表基本块入口；

    3. 用法：“我们只定义规则和事实，分析过程有Datalog引擎自动完成”

    4. 考的比较灵活，要理解。考试中可能出现新定义的谓词和规则

part3 · Instruction Scheduling

  1. 为什么要调度？

    1. 现代高性能处理器每个时钟周期可以执行多个操作

  2. 为什么编译器很重要？

    1. 硬件只能执行已经取出的指令。

      1. 处理器只能对“已经被取指单元抓取到的指令”进行操作。
      2. 如果下一条指令因为数据依赖、缓存未命中等原因卡住了，硬件就会停下来等，效率降低。

    2. 硬件缓冲资源有限

      1. 这些缓冲区空间非常有限，一旦塞满，就不得不停止取指、停止执行

    3. 编译器可以通过指令调度提升效率

      1. 译器分析指令之间的依赖关系，可以把无依赖的指令排在一起，尽量避免空等。
      2. 这使得处理器可以在一个周期里执行多个指令，提高硬件利用率
